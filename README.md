# backup-sh

A simple backup system for Ubuntu VPS: a server receives file uploads from one or more clients over HTTP. Clients are identified by connection IP. The server keeps files per client until total size exceeds a limit, then deletes oldest files when new uploads arrive.

## Requirements

- **Server**: `bash`, `socat` (`apt install socat`), standard GNU utils
- **Client**: `bash`, `curl`

## Project layout

```
server/
  setup.sh         # Prompt for env, install socat, write server.env
  start.sh         # Start the backup server
  stop.sh          # Stop the server (kill process on PORT)
  server.env.template
  server.env       # Generated by setup.sh (gitignored)

client/
  setup.sh         # Prompt for env, install curl, write client.env
  send.sh          # Upload file or directory to server
  client.env.template
  client.env       # Generated by setup.sh (gitignored)
```

## Quick start

**On the backup server (receiver)**

1. `cd server && bash setup.sh` — prompts for each variable and installs socat if missing (use `-f` to overwrite existing `server.env`).
2. `bash start.sh` — start the server (runs in foreground; for background, run `nohup bash start.sh &` or use a process manager).

To stop: `bash stop.sh` (kills the process listening on the port from `server.env`).

**On a client VPS**

1. `cd client && bash setup.sh` — prompts for env and installs curl if missing.
2. Upload: `bash send.sh [server_url] <file_or_dir> [--key-file PATH] [--insecure]`
   - If `SERVER_URL` is set in `client.env`, you can run: `bash send.sh /path/to/file-or-dir`

## Configuration

Templates live in each directory: `server/server.env.template`, `client/client.env.template`. The setup scripts prompt for each variable (Enter keeps the default) and write `server.env` / `client.env`. Do not commit `*.env` files; they are in `.gitignore`.

### Server (`server/server.env`)

| Variable | Default | Description |
|----------|---------|-------------|
| `BACKUP_ROOT` | `/var/backups/incoming` | Directory for all client backups (one subdir per client IP). |
| `PORT` | `9999` | TCP port to listen on. |
| `ACCESS_KEY` | (required) | Secret key; must match the key used by clients. |
| `MAX_SIZE_PER_CLIENT` | `1G` | Max total bytes per client (e.g. `1G`, `500M`). Oldest files deleted when exceeded. |
| `LOG_FILE` | `/var/log/backup-server.log` | Server log path. |

### Client (`client/client.env`)

| Variable | Default | Description |
|----------|---------|-------------|
| `SERVER_URL` | (optional) | Default server URL (e.g. `http://backup-server:9999`). Can override as first arg to send.sh. |
| `ACCESS_KEY` | (optional) | Secret key; if empty, uses `KEY_FILE` or `BACKUP_ACCESS_KEY` env. |
| `KEY_FILE` | `~/.backup-sh-key` | Path to file containing the access key. |
| `LOG_FILE` | (empty) | If set, client logs are also appended to this file. |

## Server

- **start.sh** — Starts `socat` and handles each connection (handler mode). Backups under `BACKUP_ROOT/<client_ip>/`. When a client exceeds `MAX_SIZE_PER_CLIENT`, oldest files are deleted; then the new file is written. Logs to `LOG_FILE` and stderr when in foreground.
- **stop.sh** — Stops the server by killing the process listening on `PORT` (from `server.env`).

## Client

- **send.sh** — Usage: `bash send.sh [server_url] <file_or_dir> [--key-file PATH] [--insecure]`. Directories are sent recursively. Logs to stderr and optionally to `LOG_FILE` if set in `client.env`.

## Security

- Server accepts uploads only when the `X-Access-Key` header matches `ACCESS_KEY`.
- Client is identified by **connection source IP**; file paths are sanitized to prevent path traversal.

## Gitignore

`server/server.env` and `client/client.env` are ignored so secrets are not committed.
